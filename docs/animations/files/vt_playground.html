The Vt distribution playground

Every Chapter 3 failure mode is "the bells move"; every fix is "cope with the bells moving." Drag the physics and watch.

state: erased (Vt low → reads as "1")

Program pushes electrons through the oxide into the floating gate (Fowler–Nordheim tunneling) — trapped charge raises the cell's threshold voltage. Erase pulls them back out, block-wide. Each pass slightly damages the oxide: that damage is endurance, and scene 3's P/E slider is this scene repeated 3,000 times. (Figs 3-1 / 3-2 / 1-9)

each cell lands slightly differently — watch the bell emerge

The distribution isn't drawn — it emerges. Programming is a statistical process (ISPP pulses, cell variation), so a population of identically-programmed cells lands in a bell around the target Vt. Every "distribution diagram" in the book is population statistics, not a curve someone chose. (Fig 3-3's real meaning)

RBER ≈ 0.0e-0

P/E cycles widen and drift the bells (oxide wear — Figs 3-46/3-48/3-53). Retention leaks charge: bells drift left (Fig 3-54). Reads nudge neighbors: bells drift right (Figs 3-44/3-55). Red glow = overlap = cells that misread against the fixed dashed references. Read Retry (Fig 3-57) walks the references to the new valleys — errors collapse as long as the bells haven't merged: that clause is the entire limit of hard-decision reading. Soft reads add extra strobes in the overlap and give each cell a confidence (LLR) — the gray zone is exactly what LDPC consumes.
