Power-loss rebuild & snapshots

The map table lives in RAM. RAM dies with the power. What saves you is that every flash page secretly carries its own passport.

Each written page carries metadata: its LBA + a timestamp (Fig 4-43). After a crash, a full-scan rebuild crawls every page re-deriving the map; when the same LBA appears twice, the timestamp duel keeps the newer one (Fig 4-44). Snapshots (Fig 4-45) persist the map periodically — the next rebuild loads the snapshot and scans only the tail written after it (Fig 4-46). Try yanking mid-run with snapshots on and off and compare the rebuild times.
