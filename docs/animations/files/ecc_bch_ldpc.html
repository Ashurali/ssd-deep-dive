Stronger codes in action: BCH & LDPC

Hamming pointed at one error because its syndrome literally spelled the address. Real flash flips many bits at once, so drives use codes that either solve for several error positions algebraically (BCH) or let bits and checks gossip until they agree (LDPC). Both are running live below.

Hamming · 1 error
Syndrome = the error's position, in binary. One shot, one bit. The foundation.
BCH · t errors
Several syndromes → solve for an error-locator polynomial → search for its roots. Fixes many bits at once.
LDPC · many errors
A sparse graph where checks and bits pass messages and iterate to a valid word. Scales to huge codewords + soft info.

LDPC — message passing on a Tanner graph

phase: ready

No single formula. 12 bit-nodes (circles) connect to 9 check-nodes (squares); each check watches a handful of bits. A check goes red when its bits have odd parity. Every round, bits look at how many of their checks are unhappy and flip if outvoted — the graph gossips its way back to a valid codeword. (This 12-bit graph is a scaled-down stand-in; real LDPC codes are thousands of bits.)

Step · ready

The all-zero word is a valid codeword. Flip a bit (click a circle or use the button) and press Run decoding — watch the red checks vote it back.

Each round has two half-steps: checks report which bits look wrong, then bits flip if enough checks complain.

How the toys scale to a real drive

Same ideas, far bigger. A production ECC codeword is ~1–4 KB, not 12–15 bits, and each parity check touches only a handful of them — that's the "low density" in LDPC. The scaled-up machines correct tens of bit-flips per codeword, and that headroom is exactly what lets flash keep shrinking cells (MLC → TLC → QLC), where the raw error rate climbs.

BCH

t errors · solve + search

Real BCH works in fields like GF(2¹³⁺) over KB-sized words. The Chien search you watched above runs as dedicated silicon, testing every position in parallel. Long the SSD workhorse for its predictable, guaranteed t-bit correction.

LDPC

many errors · soft messages

The real win: production LDPC passes probabilities (how confident is each bit?), not hard 0/1 votes — it reads the analog voltage margin the flash reports. That soft belief-propagation corrects far more than the hard bit-flipping shown here, getting close to the theoretical limit. Why it's now mainstream for dense TLC/QLC.
