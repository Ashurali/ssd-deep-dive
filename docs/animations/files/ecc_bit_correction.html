A flipped bit announces its own location

Flash storage silently corrupts bits. The trick of ECC isn't just noticing an error — it's arranging the extra "parity" bits so that the pattern of failed checks spells out exactly where the error is, in binary. Watch it happen below with a Hamming code.

1 · Detect
One parity bit over everything tells you an odd number of bits flipped — but not which one. You can catch the error, not fix it.
2 · Locate
Use several parity bits, each covering a different overlapping group. A single flip breaks a unique combination of checks — like coordinates for the error.
3 · The trick
Choose the groups so the failing checks, read as a binary number, equal the error's position. No searching — the math points straight at it.

From this toy to what's inside your SSD

The Hamming code above fixes one bit per codeword — perfect for seeing the mechanism, too weak for real flash. Production drives keep the same spirit (structured parity → a syndrome that locates errors) but scale it up massively so they can correct many bit-flips at once. That headroom is exactly what lets flash keep getting denser.

BCH

t errors · algebraic

Generalizes Hamming using Galois-field polynomial algebra. Its syndrome feeds an error-locator polynomial that pins down several flipped bits at once. Long the SSD workhorse.

LDPC

many errors · probabilistic

A sparse check matrix where bits and checks pass "belief" messages back and forth, iterating toward a valid codeword — and it can use the analog voltage margin, not just hard 0/1. Corrects the most per bit of overhead, which is why it's becoming mainstream for dense TLC/QLC.
