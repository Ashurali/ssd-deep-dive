Stripe RAID & the chained warships

Four data dies + one XOR parity die. Losing a die is survivable — but the stripes are chained together like the warships at Red Cliffs, and GC feels the chain.

Every stripe: D0 ⊕ D1 ⊕ D2 ⊕ D3 = P, so any one member can be recomputed from the other four — watch the XOR run bit by bit. But that equation is also a chain: garbage-collect one member and the parity no longer covers the stripe, so the FTL must move (or re-parity) the whole stripe together. And when a half-filled stripe must close, the gap is padded with dummy data — write amplification you paid purely for protection. (Figs 3-58/3-59)
